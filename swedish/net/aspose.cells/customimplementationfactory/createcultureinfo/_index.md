### Anmärkningar

Den här implementeringen är användbar för situationer: 1. Vissa kulturer kanske inte stöds av användarens miljö och att skapa den nödvändiga CultureInfo med given identifierare kan orsaka Exception. För att undvika undantaget kan användaren åsidosätta denna metod för att tillhandahålla en giltig CultureInfo-instans för unsupported one. 2. Användaren kanske vill ange några anpassade egenskaper för vissa kulturer för att få förväntat resultat för formatering. För detta ändamål kan användaren åsidosätta denna metod för att tillhandahålla CultureInfo-instansen med användarspecificerade egenskaper. Observera UseUserOverride-egenskapen för returnerad CultureInfo-instans kan påverka det formaterade resultatet. Om det är falskt kan vissa egenskaper hos den returnerade CultureInfo-instansen åsidosättas av vår inbyggda formateringsmotor enligt formateringskraven för olika scenarier. Om det är sant kommer vi inte att göra det ändra alla egenskaper för den och använd den för att formatera värden direkt. Så, om användaren har spe cifierade anpassade egenskaper för den returnerade CultureInfo-instansen, se till att dess UseUserOverride är sant.
