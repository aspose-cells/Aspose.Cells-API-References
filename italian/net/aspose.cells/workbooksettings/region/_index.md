## WorkbookSettings.Region property

Ottiene o imposta le impostazioni internazionali per la cartella di lavoro.

```csharp
public CountryCode Region { get; set; }
```

### Osservazioni

1. Impostazioni regionali utilizzate dal componente Aspose.Cells per una cartella di lavoro caricata dal file modello: i). Per un file XLS, ci sono campi definiti per le impostazioni regionali e MS Excel salva i dati delle impostazioni regionali nel file durante il salvataggio del file XLS. Quindi, utilizziamo la regione salvata nel file modello per la cartella di lavoro. Se non lo fai desidera utilizzare la regione salvata nel file XLS, ripristina quella prevista (come CountryCode.Default) dopo aver caricato il file modello. E salviamo anche il valore specificato dall'utente (con questo metodo) nel file quando si salva un file XLS. ii). Per altri formati di file, come XLSX, XLSB... ecc., non esiste alcun campo definito per le impostazioni regionali nella specifica del formato file. Quindi, utilizziamo le impostazioni regionali dell'ambiente dell'applicazione per la cartella di lavoro. E, il il valore specificato dall'utente (con questo metodo) non può essere mantenuto per i file generati con quei formati di file. 2. Per l'effetto di visualizzazione in MS Excel: Le impostazioni regionali applicate qui possono avere effetto solo in fase di esecuzione con il componente Aspose.Cells e non durante la visualizzazione del file generato con MS Excel. Anche per il file XLS generato in cui sono stati salvati i dati delle impostazioni regionali specificati, durante la visualizzazione/modifica con MS Excel, la regione utilizzata per eseguire la formattazione con MS Excel è sempre quella predefinita impostazioni regionali dell'ambiente in cui è in esecuzione MS Excel, non quello salvato nel file. È il comportamento di MS Excel e non può essere modificato dal codice.
