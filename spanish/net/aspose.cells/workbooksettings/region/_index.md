## WorkbookSettings.Region property

Obtiene o establece la configuración regional para el libro de trabajo.

```csharp
public CountryCode Region { get; set; }
```

### Observaciones

1. Configuración regional utilizada por el componente Aspose.Cells para un libro de trabajo cargado desde el archivo de plantilla: i). Para un archivo XLS, hay campos definidos para la configuración regional y MS Excel guarda los datos de configuración regional en el archivo al guardar el archivo XLS. Por lo tanto, usamos la región guardada en el archivo de plantilla para el libro de trabajo. Si no lo hace desea usar la región guardada en el archivo XLS, restablézcala a la esperada (como, CountryCode.Default) después de cargar el archivo de plantilla. Y también guardamos el valor especificado por el usuario (mediante este método) en el archivo al guardar un archivo XLS. ii). Para otros formatos de archivo, como XLSX, XLSB, etc., no hay un campo definido para la configuración regional en la especificación de formato de archivo. Por lo tanto, usamos la configuración regional del entorno de la aplicación para el libro de trabajo. Y, el el valor especificado por el usuario (mediante este método) no se puede mantener para los archivos generados con esos formatos de archivo. 2. Para el efecto de vista en MS Excel: La configuración regional aplicada aquí puede tener efecto solo en tiempo de ejecución con el componente Aspose.Cells y no al ver el archivo generado con MS Excel. Incluso para el archivo XLS generado en el que se han guardado los datos de configuración regional especificados, al verlo/editarlo con MS Excel, la región utilizada para realizar el formateo por MS Excel siempre es la predeterminada configuración regional del entorno donde se ejecuta MS Excel, no la guardada en el archivo. Es el comportamiento de MS Excel y no se puede cambiar por código.
