## DBConnection.SeverCommand property

Especifica una segunda cadena de texto de comando que se mantiene cuando los campos de página basados en el servidor de tabla dinámica están en uso. Para las conexiones ODBC, serverCommand suele ser una consulta más amplia que el comando (ninguna cláusula WHERE está presente en la primera). En función de estos 2 comandos (Command y ServerCommand), se puede completar la interfaz de usuario del parámetro y se pueden construir consultas parametrizadas

```csharp
public string SeverCommand { get; set; }
```
