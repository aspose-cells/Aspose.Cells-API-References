## WorkbookSettings.Region property

Obtient ou définit les paramètres régionaux du classeur.

```csharp
public CountryCode Region { get; set; }
```

### Remarques

1. Paramètres régionaux utilisés par le composant Aspose.Cells pour un classeur chargé à partir du fichier modèle : i). Pour un fichier XLS, des champs sont définis pour les paramètres régionaux et MS Excel enregistre les données des paramètres régionaux dans le fichier lors de l'enregistrement du fichier XLS. Nous utilisons donc la région enregistrée dans le fichier de modèle pour le classeur. Si vous ne le faites pas souhaitez utiliser la région enregistrée dans le fichier XLS, veuillez la réinitialiser à celle attendue (par exemple, CountryCode.Default) après avoir chargé le fichier de modèle. Et nous enregistrons également la valeur spécifiée par l'utilisateur (par cette méthode) dans le fichier lors de l'enregistrement d'un fichier XLS. ii). Pour les autres formats de fichiers, tels que XLSX, XLSB... etc., aucun champ n'est défini pour les paramètres régionaux dans la spécification du format de fichier. Nous utilisons donc les paramètres régionaux de l'environnement de l'application pour le classeur. Et, le la valeur spécifiée par l'utilisateur (par cette méthode) ne peut pas être conservée pour les fichiers générés avec ces formats de fichier. 2. Pour l'effet de vue dans MS Excel : Les paramètres régionaux appliqués ici ne peuvent prendre effet qu'au moment de l'exécution avec le composant Aspose.Cells et non lors de la visualisation du fichier généré avec MS Excel. Même pour le fichier XLS généré dans lequel les données de paramètres régionaux spécifiés ont été enregistrées, lors de la visualisation/modification avec MS Excel, la région utilisée pour effectuer le formatage par MS Excel est toujours la valeur par défaut paramètres régionaux de l'environnement dans lequel MS Excel est exécuté, et non celui enregistré dans le fichier. C'est le comportement de MS Excel et ne peut pas être modifié par le code.
