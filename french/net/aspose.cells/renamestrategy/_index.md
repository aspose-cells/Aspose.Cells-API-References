## RenameStrategy enumeration

Option de stratégie pour les noms de colonnes en double.

```csharp
public enum RenameStrategy
```

### Valeurs

| Nom | Évaluer | La description |
| --- | --- | --- |
| Exception | `0` | Lève une exception. |
| Digit | `1` | Nommé avec un chiffre. Les noms en double deviendront ...1, ...2, etc. |
| Letter | `2` | Nommé avec une lettre.. Les noms dupliqués deviendront ...A, ...B, etc. |

### Remarques

Lors du traitement de données avec des en-têtes, certains scénarios exigent que les en-têtes ne soient pas dupliqués pour toutes les colonnes. Par exemple, lors de l'exportation de données vers une table de données et que l'en-tête doit être pris comme nom de colonne de la table de données, les valeurs dupliquées de l'en-tête ne sont pas valides . Pour ce genre de situations, l'utilisateur peut déterminer comment les gérer en spécifiant cette stratégie.
