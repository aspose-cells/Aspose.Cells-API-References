## DBConnection.SeverCommand property

Spécifie une deuxième chaîne de texte de commande qui est conservée lorsque les champs de page basés sur le serveur de tableau croisé dynamique sont utilisés. Pour les connexions ODBC, serverCommand est généralement une requête plus large que command (aucune clause WHERE n'est présente dans la première). Sur la base de ces 2 commandes (Command et ServerCommand), l'interface utilisateur du paramètre peut être remplie et des requêtes paramétrées peuvent être construites

```csharp
public string SeverCommand { get; set; }
```
