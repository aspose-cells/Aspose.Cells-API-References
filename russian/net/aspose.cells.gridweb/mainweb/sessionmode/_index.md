## MainWeb.SessionMode property

Получает или устанавливает режим сеанса сетки. Существует 4 типа режима сеанса:&lt;br /&gt; 1. Сеанс (по умолчанию):Используйте системный сеанс для хранения данных листа.&lt;br /&gt; Обычно asp.net использует состояние сеанса InProc. Сетка также поддерживает состояние сеанса внешнего процесса "StateServer" и состояние сеанса SQLServer.&lt;br /&gt; 2. ViewState:используйте состояние просмотра страницы для хранения данных листа.&lt;br /&gt; 3. Пользовательский:используйте события LoadCustomData и SheetDataUpdated для сохранения/восстановления данных листа.&lt;br /&gt; 4. Файл:сохранение/восстановление данных листа в SessionStorePath.&lt;br /&gt;

```csharp
public SessionMode SessionMode { get; set; }
```

### Примечания

При использовании SessionMode.ViewState сетка будет хранить данные листов в состоянии просмотра страницы. Это уменьшит использование памяти сервером , но размер страницы будет больше, и это повлияет на общую производительность.

### Примеры

```csharp
[C#]
	GridWeb1.SessionMode = SessionMode.ViewState;
	
[Visual Basic]
	GridWeb1.SessionMode = SessionMode.ViewState
```
