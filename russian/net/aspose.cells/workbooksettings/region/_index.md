## WorkbookSettings.Region property

Получает или задает региональные настройки для книги.

```csharp
public CountryCode Region { get; set; }
```

### Примечания

1. Региональные настройки, используемые компонентом Aspose.Cells для рабочей книги, загруженной из файла шаблона: i). Для файла XLS есть поля, определенные для региональных настроек, и MS Excel сохраняет данные региональных настроек в файл при сохранении файла XLS. Таким образом, мы используем сохраненный регион в файле шаблона для рабочей книги. хотите использовать регион, сохраненный в файле XLS, сбросьте его на ожидаемый (например, CountryCode.Default) после загрузки файла шаблона. И мы также сохраняем указанное пользователем значение (этим методом) в файл при сохранении файла XLS. ii). Для других форматов файлов, таких как XLSX, XLSB и т. д., в спецификации формата файла не определено поле для региональных настроек. Итак, мы используем региональные настройки среды приложения для книги. И, указанное пользователем значение (данным методом) не может быть сохранено для сгенерированных файлов с этими форматами файлов. при просмотре сгенерированного файла в MS Excel. Даже для сгенерированного XLS-файла, в котором сохранены указанные данные региональных настроек, при просмотре/редактировании его с помощью MS Excel, используемый регион для выполнения форматирования в MS Excel всегда является значением по умолчанию региональные настройки среды, в которой работает MS Excel, не та, что сохранена в файле. Это поведение MS Excel, и его нельзя изменить с помощью кода.
